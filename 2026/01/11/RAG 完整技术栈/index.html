RAG 完整技术栈

1. 采集 & 预处理

内容抓取

  • 网页爬虫
  • API 同步
  • 数据库 CDC
  • SaaS 连接器

格式归一

  • PDF/Doc → Markdown
  • OCR & 表格还原
  • PPT 大纲提取

清洗

  • 去噪（页眉页脚、广告）
  • 去重
  • PII 脱敏

元数据注入

  • 标题/时间/作者
  • 权限/版本号
  • chunk 来源坐标

2. 深度内容理解

层级切分

  • 句子 → 段落 → 章节
  • 语义滑窗 overlap

多粒度摘要

  • 句子级摘要
  • 段落级摘要
  • 文档级摘要

多模态解析

  • 图表 caption 生成
  • 图像 OCR
  • 音频转文本

知识图谱

  • 实体抽取
  • 关系抽取
  • 多跳问答

3. 索引策略

混合索引

  • 稠密向量（embedding）
  • 稀疏向量（BM25/SPLADE）
  • 倒排关键词

多级索引

  • 文档级粗排
  • chunk 级精排
  • HNSW → PQ

结构化字段

  • 时间/权限/标签
  • 布尔过滤

动态更新

  • CDC Upsert
  • 软删除 & 版本回滚

4. 查询理解

Query 改写

  • 同义词扩展
  • 拼写纠错
  • Query2Doc / HyDE

意图识别

  • FAQ 路由
  • 表格/工单路由

多轮会话

  • 指代消解
  • 上下文补齐

5. 检索策略

多路召回

  • 向量 + 关键词 + KG + SQL

精排模型

  • Cross-encoder
  • ColBERT / Listwise

检索量自适应

  • 动态 top-k
  • 置信度阈值
  • Token 成本控制

6. 上下文组装

Rerank 截断

  • 相关性分数
  • Token 预算

上下文压缩

  • 抽取式摘要
  • LLM 概要
  • 句子压缩

引用溯源

  • [doc_id] 标注
  • [page:line] 坐标

7. 生成增强

Prompt 模板

  • Few-shot / CoT
  • Self-Ask / Tool-use

可控生成

  • JSON Schema
  • Function Call

事实校验

  • Self-consistency
  • NLI 幻觉检测

多模态输出

  • 表格/图表/图片 URL

8. 系统 & 运维

评测

  • RAGAs / TruLens
  • Context Recall / Precision
  • Answer Similarity

A/B 实验

  • embedding 模型
  • chunk_size / top-k

缓存

  • 语义缓存
  • KV 缓存

权限 & 合规

  • 行级/列级 ACL
  • 加密 & 审计日志

弹性伸缩

  • embedding 批量服务
  • 向量库分片
  • GPU/CPU 混布

RAG什么是RAG? RAG（Retrieval-Augmented Generation，检索增强生成）一种将“信息检索”与“生成式大模型”耦合的架构。系统先通过稠密向量或稀疏检索，从大规模外部知识库中定位与当前查询最相关的文档片段（context），再把这些片段连同原始查询一起送入大语言模型（LLM）作为 prompt 的上下文，让模型在生成回答时既利用内部参数知识，又利用最新的外部知识，从而显著降低幻觉、提升时效性与可溯源性。其典型流程：Indexing → Retrieval → Augmentation → Generation。一句话解释就是：先用“搜索引擎”找到相关文档，再让“作文高手”参考这些资料写答案，既新鲜又靠谱。 实现方式 1.1平台实现 fastgpt、coze、dify等 1.2代码实现 代码有哪些技术栈？ 具体技术细节有哪些？ 文档提取 1.1txt问本提取 1.2pdf提取工具 1.3表格提前工具 使用Llama Hub中的Database Reader工具连接MySQL数据库，并将数据库表数据加载到Llama Index的Documents...
