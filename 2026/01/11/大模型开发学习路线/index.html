大模型开发学习路线

一、岗位能力地图（你需要会到什么程度）

  • 编程与工程
    • Python/TypeScript，异步与并发，API 设计，单元/集成测试，Docker，基础云原生（K8s/Helm）。
  • LLM 基础与调用
    • token/上下文/温度/采样，系统提示与模式约束（JSON schema）、函数/工具调用，多模型路由与回退。
  • RAG 与检索
    • 数据清洗与切块（chunk）、embedding 选型，混合检索（BM25+向量）、重排（rerank）、重复消解，召回评测。
  • 微调与蒸馏
    • SFT/LoRA/QLoRA、指令数据构造、对齐（RLAIF/RLHF 基本概念）、量化（INT8/4）与部署兼容性。
  • 评测与治理
    • 黄金集构建、离线评测（检索/回答/安全）、线上反馈闭环、A/B、可观测（日志/追踪）、版本化与回滚。
  • 推理与部署
    • vLLM/TGI/Ollama，连续批处理、KV cache、流式、超时与重试、熔断、限流、弹性扩缩。
  • 安全与合规
    • 越狱/提示注入防护、PII 脱敏、内容审核、权限与审计；国内合规（等保、数据安全法、个保法）意识。
  • 多模态与文档理解
    • OCR/版面解析（PDF/表格/截图）、语音 ASR/TTS、图文混合输入。
  • 产品与ROI
    • 需求拆解、指标体系、单位经济核算、上线灰度与推广。

二、16 周学习路线（可压缩/拉伸）
第1–4周：打基础 + 最小可用产品

  • 目标：能独立做“文档问答 + 结构化输出 + 工具调用”的端到端 Demo，并接入日志。
  • 学习要点
    • LLM API 与函数调用；提示模板与JSON模式约束；流式输出。
    • 向量库与RAG基础：切块、embedding、BM25/向量/混合检索、简单重排。
    • 基础工程：异步、超时/重试、速率限制、提示/数据版本化、日志与追踪。
  • 实作任务
    • 做一个企业文档问答（本地PDF/网页），支持引用出处、JSON 结构化摘要、函数调用检索。
    • 建10–30条黄金问题，记录R@K/答案通过率与延迟。
  • 验收标准
    • R@10≥0.75，答案通过率≥65%，P95延迟≤3s（云模型），错误率<1%。

第5–8周：评测治理 + 体验成本优化

  • 学习要点
    • 离线评测：检索与生成分开评；LLM-as-judge；A/B 测试；观测平台（Langfuse/自建）。
    • 质量优化：混合检索+重排（bge/cohere/jina rerank）、去重/归并、查询改写，记忆与对话状态。
    • 成本性能：语义缓存、批处理、模型路由/回退（大→小）、多供应商抽象。
    • 安全：越狱与注入测试、PII 脱敏、规则与阈值。
  • 实作任务
    • 给第一个项目接入评测流水线与缓存/回退；出一页“质量-延迟-成本”曲线与取舍说明。
  • 验收标准
    • 答案通过率提升≥10pp；P95 延迟下降≥30%；单请求成本下降≥30%；安全用例通过率≥95%。

第9–12周：微调与开源部署

  • 学习要点
    • 数据集构造与清洗；SFT/LoRA/QLoRA；小模型蒸馏；评测对比“仅RAG vs RAG+微调”。
    • 推理服务：vLLM/TGI，连续批、KV cache、量化；灰度与回滚。
  • 实作任务
    • 选7B–14B开源模型，对你的场景做小规模LoRA（5k–50k样本）；用vLLM部署推理对外服务。
  • 验收标准
    • 在黄金集上，相比仅RAG：关键指标显著提升（>5–10pp）或在同等质量下成本/延迟更优；QPS≥20（视模型与硬件）。

第13–16周：垂直场景 MVP + 上线硬化

  • 选题：客服质检、合同审阅、代码辅助、售前问答、运营文案等。
  • 学习要点
    • 多模态（PDF/OCR/截图）、权限与租户隔离、审计追踪、告警与SLO、容灾。
  • 实作任务
    • 打磨可在线演示的 MVP，接入真实小规模用户；出版本迭代节奏与Roadmap。
  • 验收标准
    • 端到端：召回/准确/安全/延迟/成本指标齐全；能演示故障回退与审计；完成一轮用户反馈闭环。

三、必备工具与选型（保持“可替换”）

  • 模型与供应商
    • 国际：OpenAI、Anthropic、Google、Mistral；开源：Llama 3.x、Mistral、Qwen2/2.5、Baichuan、Yi 等。
    • 国内：阿里通义（DashScope/Qwen）、百度千帆（文心）、腾讯混元、讯飞星火、百川等。
  • 开发与编排
    • 通用：Python/TS、FastAPI/Express、Pydantic。
    • 编排/Agent：LangChain、LlamaIndex、DSPy（或自研轻量DAG/状态机）。
  • 向量检索
    • FAISS、pgvector/Postgres、Milvus/Weaviate；文本检索：BM25（Elasticsearch/Opensearch）。
    • Embedding：bge-m3、e5-large、或供应商 embedding。
    • 重排：bge-reranker、Cohere/Jina Rerank。
  • 观察与评测
    • 观测：Langfuse、OpenTelemetry+ELK/ClickHouse。
    • 评测：Ragas、TruLens、DeepEval、自建黄金集与判分器。
  • 部署与推理
    • vLLM、TGI、Ollama（本地）、KServe/Ray Serve；容器：Docker/K8s；监控：Prometheus/Grafana。
  • 数据处理/文档理解
    • 解析：Unstructured、llama-parse、Textract；OCR：PaddleOCR/Tesseract；表格/版面：layoutparser。

四、作品集路线（面试最有说服力）

  • 项目1：企业知识库RAG
    • 指标：R@10、nDCG、答案通过率、引用正确率、P95延迟、成本/请求、缓存命中率。
  • 项目2：工具调用工作流
    • 工具：检索/SQL/日程/第三方API；有状态编排与回退；显示“不可观测的Agent”对比“显式DAG”的收益。
  • 项目3：垂直场景MVP（任选其一）
    • 合同条款抽取与风险提示（结构化JSON+证据页码）、客服质检（违规检测+评分）、代码评审（issue 建议+打 patch）。
  • 可选：微调小模型
    • 用 LoRA/QLoRA 做 7B 指令微调，和“仅RAG”做消融对比。

五、必须掌握的评测指标（拿来即用）

  • 检索：Recall@K、nDCG、MRR、查询覆盖率、重复率。
  • 生成：LLM-as-judge 的 helpfulness/groundedness、事实一致性、引用命中率；必要时ROUGE/Exact Match。
  • 系统：P50/P95/P99 延迟、QPS、错误率、重试率、回退率、缓存命中率、工具成功率。
  • 成本：tokens/请求、单位成本、缓存节省%、不同模型路由占比。
  • 安全：注入/越狱通过率、敏感词/PII 识别率、拒答准确率。

六、面试与求职准备

  • 文档化你的取舍：为什么选混合检索、何时用重排/微调、如何做回退与容错、如何降本提速。
  • 一页架构图+一页指标表+一页评测方法（含黄金集样例与抽样误差）。
  • 准备系统设计题：设计企业RAG/代码助手/内容审核流水线；讲清数据、评测、SLO 与运维。
  • 代码题：JSON 模式约束、函数调用、检索重排、并发与超时。

七、强力加分项（区分度高）

  • 评测与治理体系落地
    • 会构造高质量黄金集、搭 LLM-as-judge、线上反馈闭环与自动回归测试。
  • 推理与成本优化
    • vLLM 连续批、KV cache 命中优化、语义缓存、分层路由；能做单位经济核算与容量规划。
  • 安全与合规
    • 对越狱/注入有实战对抗策略；PII/涉敏数据流全链路脱敏与审计；国内等保/数据出境意识。
  • 多模态与复杂文档
    • OCR+版面解析+RAG，能处理表格/截图/扫描PDF 并给出稳定的证据定位。
  • 小模型能力
    • LoRA/QLoRA/量化部署、蒸馏到小模型或规则混合以提升毛利。
  • 平台化能力
    • 多模型抽象层、灰度/金丝雀、SLO/告警、K8s+KServe/Ray、私有化部署。
  • 业务洞察与沟通
    • 能把技术指标转译为业务价值与ROI；写清晰的设计文档；跨团队协作推进落地。
  • 开源贡献/技术影响力
    • 在 HF/LangChain/Ragas 等提PR或发布小工具；技术博客/分享。

八、建议的学习资源（任选其一条线深入）

  • 体系化
    • Hugging Face Transformers/TRL 文档
    • LangChain/LlamaIndex 官方教程
    • vLLM/TGI 文档与示例
  • 论文与要点
    • Attention Is All You Need、InstructGPT/RLHF 概念、Lost in the Middle（长上下文）、Toolformer/Function Calling、RAG 系列综述。
  • 实战博客/仓库
    • RAGas/TruLens 示例、Milvus/pgvector 实战、DSPy 提示最优化。

学习AI（人工智能）

学习AI（人工智能）是一个系统性过程，需要结合数学基础、编程能力、理论知识、实践项目和持续跟进前沿。以下是分阶段的详细学习路径，适合零基础或有一定基础的学习者参考：

一、打牢基础：数学与编程

AI的核心是“用数学模型解决问题”，编程是实现模型的工具。这一步是基石，不可跳过。

1. 数学基础（核心！核心！核心！）

AI涉及大量数学推导，至少需掌握以下核心领域：

  • 线性代数：向量、矩阵运算（加减乘、转置、逆、特征值/特征向量）、空间变换（如PCA降维的原理）。
    • 推荐资源：MIT 18.06线性代数课（B站/网易公开课）、《线性代数及其应用》（Gilbert Strang）。
  • 微积分：导数、偏导数、梯度下降原理（AI优化的核心）、多变量函数极值。
    • 推荐资源：3Blue1Brown《微积分的本质》（B站可视化讲解，直观理解）、《普林斯顿微积分读本》。
  • 概率与统计：概率分布（正态分布、二项分布）、期望/方差、最大似然估计、贝叶斯定理（朴素贝叶斯、贝叶斯网络的基础）。
    • 推荐资源：《深入浅出统计学》（适合零基础）、Coursera《概率统计入门》（吴恩达）。

2. 编程基础（工具！工具！工具！）

AI主流工具是Python（生态最完善，库最多），需掌握：

  • Python基础：语法（变量、循环、函数、类）、数据结构（列表、字典、数组）。
    • 推荐资源：菜鸟教程（快速入门）、B站“黑马程序员Python基础”（系统全面）。
  • 数据处理与可视化库：
    • NumPy（矩阵运算，AI的“数学计算器”）、Pandas（数据清洗/分析，处理表格数据）、Matplotlib/Seaborn（画图，可视化数据和模型结果）。
    • 推荐练习：用Pandas处理CSV数据（如Kaggle的Titanic数据集），用Matplotlib画折线图/散点图。
  • 开发工具：Jupyter Notebook（交互式编程，适合调试和展示）、Anaconda（管理Python环境和库）、Google Colab（免费GPU，无需本地配置）。

二、核心理论：机器学习（ML）

机器学习是AI的基础，是“让机器从数据中学习规律”的方法论。先掌握经典算法，再学深度学习。

1. 机器学习入门（理解“学习”的本质）

先搞懂核心概念：

  • 监督学习（有标签数据：预测房价、识别猫/狗）、无监督学习（无标签数据：用户分群、异常检测）、强化学习（试错学习：AlphaGo下棋）。
  • 模型评估：准确率、精确率、召回率、F1-score、过拟合/欠拟合（如何用正则化、交叉验证解决）。

2. 经典算法（逐个攻破，理论+代码）

按“简单→复杂”顺序学习，每个算法先理解数学原理，再用代码实现（调包+手动推导）：

  • 监督学习：
    • 线性回归（预测连续值，如房价）、逻辑回归（二分类，如垃圾邮件识别）→ 理解梯度下降优化过程。
    • 决策树、随机森林（可解释性强，处理分类/回归）→ 理解信息熵、基尼系数。
    • SVM（支持向量机，小样本分类）→ 理解核函数（将低维数据映射到高维）。
    • 集成学习（AdaBoost、XGBoost，工业界常用）→ 理解“弱分类器集成变强”。
  • 无监督学习：
    • K-Means（聚类，用户分群）→ 理解距离度量（欧氏距离）、肘部法则选K值。
    • PCA（主成分分析，降维可视化）→ 理解协方差矩阵、特征值分解。

3. 学习资源（系统课程+教材）

  • 课程：吴恩达《Machine Learning》（Coursera经典课，数学推导清晰，作业用Octave，但可改用Python复现）；B站“李宏毅机器学习”（通俗易懂，案例丰富）。
  • 教材：周志华《机器学习》（西瓜书，理论严谨，适合进阶）；《机器学习实战》（用Python实现算法，偏实践）。
  • 工具库：Scikit-learn（Python机器学习库，一行代码调用算法，快速验证想法）。

三、进阶：深度学习（DL）

深度学习是机器学习的分支，基于神经网络（模仿人脑神经元），擅长处理图像、文本等复杂数据。

1. 神经网络基础（从“神经元”到“网络”）

  • 感知机（单个神经元）→ 多层神经网络（前馈网络）→ 反向传播（神经网络训练的核心，用微积分求梯度更新权重）。
  • 激活函数：ReLU、Sigmoid、Tanh（解决非线性问题）。

2. 经典深度学习模型（按应用场景学）

  • 计算机视觉（CV）：
    • CNN（卷积神经网络，处理图像）→ 理解卷积层（提取边缘/纹理）、池化层（降维）、全连接层（分类）。
    • 经典模型：LeNet-5（手写数字识别）、AlexNet（ImageNet竞赛冠军，开启深度学习热潮）、ResNet（解决深层网络梯度消失，用残差连接）。
  • 自然语言处理（NLP）：
    • RNN/LSTM/GRU（处理序列数据，如文本、语音）→ 理解循环结构解决“时序依赖”，LSTM解决长序列遗忘问题。
    • Transformer（2017年革命，NLP主流架构）→ 理解自注意力机制（Self-Attention）、Encoder-Decoder结构（BERT、GPT的基础）。
  • 其他：GAN（生成对抗网络，生成逼真图像）、AutoEncoder（自编码器，降维/去噪）。

3. 深度学习框架（选一个深入，别贪多）

用框架实现模型，避免重复造轮子：

  • PyTorch（推荐初学者）：动态图，调试方便，代码直观（Facebook开发，学术圈常用）。
  • TensorFlow/Keras（工业界常用）：静态图，适合部署（Google开发，有成熟生态）。
    • 学习资源：李沐《动手学深度学习》（PyTorch版，B站视频+开源代码，边学边敲）、TensorFlow官方教程（含Colab在线练习）。

四、实践！实践！实践！（关键！）

AI是“做出来”的，光看理论永远学不会。通过项目巩固知识，积累经验。

1. 入门级项目（用公开数据集练手）

  • 数据集：Kaggle（数据竞赛平台，找“Getting Started”标签的比赛，如Titanic生存预测、房价预测）、UCI机器学习库（简单数据集）。
  • 目标：用Scikit-learn实现经典算法，用Matplotlib可视化结果，写分析报告（为什么模型效果好/差？如何改进？）。

2. 进阶项目（深度学习应用）

  • CV方向：图像分类（CIFAR-10数据集，用CNN实现）、目标检测（用YOLOv5识别图片中的物体）。
  • NLP方向：文本分类（用BERT微调判断电影评论正负）、情感分析（爬取微博数据，分析用户情绪）。
  • 工具：用PyTorch/TensorFlow复现论文模型（如简单CNN），提交到GitHub（积累作品集）。

3. 参加竞赛/开源贡献

  • Kaggle竞赛（赢了有奖金，输了也能看大佬代码）、天池（阿里系竞赛平台）。
  • GitHub：给开源项目提PR（如修复文档bug）、复现经典论文代码（如Transformer）。

五、领域深入（选方向，别贪全）

AI子领域太多，选一个感兴趣的方向深耕，成为“专才”：

  • 计算机视觉（CV）：目标检测、图像分割、自动驾驶感知。
  • 自然语言处理（NLP）：大模型微调（如用LLaMA、ChatGLM做定制化对话）、机器翻译、文本生成。
  • 强化学习（RL）：游戏AI（用OpenAI Gym环境训练小车避障）、机器人控制。
  • 推荐系统：用协同过滤/深度学习做商品推荐（如MovieLens数据集）。

六、持续跟进前沿（AI发展太快！）

AI技术迭代极快（如2023年GPT-4、2024年Sora），需保持学习：

1. 关注最新研究

  • 论文：arXiv（cs.LG机器学习、cs.CV计算机视觉方向，每天更新）、Papers With Code（论文+代码，直接看实现）。
  • 顶会：NeurIPS、ICML、ICLR（AI领域三大顶会，每年12月/7月/5月召开，关注“Best Paper”）。

2. 社区与资源

  • 博客：Medium（Towards Data Science专栏）、机器之心/量子位（中文AI资讯）。
  • 视频：YouTube（Two Minute Papers，解读前沿论文）、B站（李沐、跟李宏毅学AI）。
  • 社区：Reddit r/MachineLearning（讨论热点）、GitHub（看Star多的开源项目，如Hugging Face Transformers库）。

避坑指南（新手常犯错误）

  1. 别跳过数学：“数学太难，直接调包行不行？” → 不行！不懂梯度下降，调参永远是“碰运气”；不懂注意力机制，用GPT时不知道怎么优化。
  2. 别只看不动手：“我看完了李沐的课，应该会了吧？” → 手生=不会！代码必须自己敲，报错了就debug（Google/Stack Overflow是你的朋友）。
  3. 别贪多求快：“今天学CNN，明天学Transformer，下周搞GAN！” → 每个算法吃透（原理+代码+项目），比囫囵吞枣学10个强。

总结路径

零基础→入门：数学（线代/概率）→ Python（NumPy/Pandas）→ 机器学习（Scikit-learn+经典算法）→ 小项目（Kaggle）。
进阶→专业：深度学习（PyTorch+Transformer）→ 方向深耕（CV/NLP）→ 复杂项目（顶会论文复现）→ 跟进前沿（读论文+社区）。

AI学习是“马拉松”，但每一步都有明确的目标和反馈。坚持6个月，你就能从“小白”变成能独立做项目的AI实践者！

常见概念的理解：

微调（Fine-tuning）和对齐 （Alignment）的区别

微调 = 让模型“更擅长做什么”；
对齐 = 让模型“按人类期望的方式做”。
技术上一个重“模仿高质量示范”，一个重“优化人类偏好和约束”；应用上一个产出“领域专家”，一个产出“可信助手”。

八种现代 LLM 架构大比较

八种目前在工业界与学术圈最常被讨论的「现代 LLM 架构」可以按两条主线来归纳：

• 主线 A：Transformer 家族内部的“微观”结构演进（4 种）
• 主线 B：跳出传统 Dense Transformer 的“宏观”范式革新（4 种）

下面把八位选手一次性摆到同一张擂台，用「技术关键词 → 关键差异 → 一句话优势/劣势」的三段式速览。

──────────────────
A. Transformer 微观演进（仍在 Decoder-only 框架里做“手术”）

  1. Dense Decoder（Llama-3 / Gemma-3 / OLMo-2）
    关键词：RMSNorm + RoPE + GQA/SWA + SwiGLU
    差异：最“经典”的堆叠，参数全部激活，无稀疏化。
    一句话：易调试、易部署，但随规模线性涨成本。
  2. GQA（Grouped-Query Attention）
    Llama-3、Mistral-Small-3.1 使用。
    差异：KV 头分组，推理显存 ∝ (n_kv_heads) 而非 (n_heads)。
    一句话：把 KV-cache 砍掉 3-8×，吞吐飞起，效果几乎不掉。
  3. MLA（Multi-Head Latent Attention）
    DeepSeek-V3 首创。
    差异：先把 K/V 投影到低维 latent，再多头展开 → 进一步压缩 KV-cache 2× 以上，且比 GQA 保更多表达能力。
    一句话：KV-cache 极限压缩术，实现难度比 GQA 高。
  4. MoE-Decoder（Llama-4 / DeepSeek-V3 / Qwen3-MoE）
    关键词：Top-K 路由 + 专家并行 + 负载均衡 loss
    差异：总参数 >100 B，但每 token 只激活 5-10 %。
    一句话：用“专家稀疏”换“计算稠密”，训练贵、推理便宜。

──────────────────
B. 跳出 Dense Transformer 的宏观革新

  1. RAG-LLM（检索增强生成）
    关键词：Encoder+Retriever+Decoder 三段式，实时外挂知识库
    差异：参数不动，把问题交给“检索 + 拼接提示”。
    一句话：不改模型就能“长”出新知识，但延迟与检索质量绑定。
  2. Mixture-of-Experts-Plus（专家混合再进化：Expert Choice、Expert Parallelism-v2）
    关键词：动态专家容量、共享专家、路由表学习
    差异：解决传统 MoE 负载不均、通信爆炸。
    一句话：让 MoE 真正跑在千卡级集群而不崩，千卡效率 ↑30 %。
  3. Mamba / State-Space 序列模型
    关键词：SSM Layer 替代 Attention，O(N) 显存
    差异：彻底告别 QKV 投影，用卷积+门控状态传递。
    一句话：百万级上下文只需线性内存，但生成质量仍落后于同规模 Transformer。
  4. Long-Context Transformer（Ring-Attention + Flash-Sequencer）
    关键词：分块 Flash-Attention、梯度 checkpoint、序列并行
    差异：不改模型结构，用系统级技巧把上下文拉到 1-10 M tokens。
    一句话：工程外挂，让 Llama-3 8 B 也能一次读完整本《三体》。
